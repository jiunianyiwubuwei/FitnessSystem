# 舞蹈评分系统使用说明

## 功能概述

舞蹈评分系统允许用户上传自己的舞蹈视频，与系统中的标准舞蹈进行比对，获得动作准确度、节奏、流畅度等维度的评分。

---

## 一、后端部署

### 1. 数据库迁移

首先需要执行数据库迁移来创建新的数据表：

```bash
cd k:\FitnessSystem\后端\admin_project

# 创建迁移文件
python manage.py makemigrations fitness

# 执行迁移
python manage.py migrate
```

### 2. 新增 API 接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/dance/standards/` | GET | 获取标准舞蹈列表 |
| `/api/dance/preprocess/` | POST | 预处理标准舞蹈视频 |
| `/api/dance/evaluate/` | POST | 提交舞蹈评分 |
| `/api/dance/scores/` | GET | 获取用户评分历史 |
| `/api/dance/score/detail/` | GET | 获取评分详情 |

### 3. 添加标准舞蹈

将标准舞蹈视频放到 `media/video/dance/standards/` 目录下，然后运行预处理命令：

```bash
# 添加标准舞蹈
python manage.py preprocess_dance <视频路径> <舞蹈名称> <舞蹈类型> --difficulty 1-5

# 示例
python manage.py preprocess_dance "media/video/dance/standards/街舞基础.mp4" "街舞基础动作" "街舞" --difficulty 1

# 查看已添加的舞蹈
python manage.py list_dances
```

## 二、评分算法说明

### 1. 核心算法：DTW (动态时间规整)

系统使用 DTW 算法来对齐两个不同长度的关键点序列，这是舞蹈评分的技术基础。

### 2. 评分维度

| 维度 | 权重 | 说明 |
|------|------|------|
| 准确度 | 40% | 基于 DTW 距离计算的动作相似度 |
| 节奏 | 20% | 视频时长与标准舞蹈的匹配程度 |
| 流畅度 | 20% | 相邻帧之间动作变化的平滑程度 |
| 关键帧 | 20% | 采样关键动作节点的相似度 |

### 3. 等级划分

| 等级 | 分数区间 | 评语 |
|------|----------|------|
| S | 95-100 | 完美！堪称范本！ |
| A | 85-94 | 优秀！动作非常标准！ |
| B | 75-84 | 良好！继续加油！ |
| C | 65-74 | 合格！还需要多加练习 |
| D | 50-64 | 不合格！建议观看标准动作 |
| E | <50 | 差距较大，请认真练习基础动作 |

## 三、使用流程

### 管理员操作

1. 在 `media/video/dance/standards/` 目录放置标准舞蹈视频
2. 运行 `python manage.py preprocess_dance` 命令预处理视频
3. 系统自动提取关键点并存储到数据库

### 用户操作

1. 进入"智慧健身"页面
2. 点击"舞蹈评分"模式切换标签
3. 从列表中选择想要挑战的舞蹈
4. 上传自己的舞蹈视频
5. 系统自动分析并显示评分结果

## 五、扩展建议

### 1. 添加更多标准舞蹈

重复使用 `preprocess_dance` 命令即可添加多个标准舞蹈。建议按难度分级，方便不同水平的用户选择。

### 2. 优化评分算法

当前算法使用 DTW 进行序列比对，可以考虑：
- 引入更精细的姿态角度计算
- 添加舞蹈风格的分类匹配
- 使用机器学习模型进行动作质量评估

### 3. 添加视频对比播放

前端可以增加"标准视频 vs 用户视频"的分屏对比功能，让用户直观看到差异。

### 4. 历史趋势分析

利用评分历史数据，展示用户的进步曲线。

## 六、常见问题

### Q: 预处理命令报错"视频文件不存在"
A: 确保视频路径正确，建议使用绝对路径。

### Q: 评分结果为 0
A: 检查标准舞蹈是否已完成预处理（关键点数据是否已提取）。

### Q: 上传视频失败
A: 检查 `media/video/dance/user_uploads/` 目录是否存在，如不存在需手动创建。
